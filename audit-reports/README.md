# audit-reports/

Auto-generated by `blog-auditor`. Each audit writes a timestamped folder per `<brand>/<slug>/`.

See [`INDEX.md`](./INDEX.md) (created on first run) for the full chronological list.

Layout:

```
audit-reports/
  INDEX.md                    ← chronological index of every audit
  <brand>/
    <slug>/
      <timestamp>/
        report.html           ← human-readable audit report with critical-metric rubric
        report.json           ← machine-readable report
        iteration-0/
          full-page.html      ← post HTML at start of iteration
          findings.json       ← findings for this iteration
          scores.json         ← layer scores + delta
        iteration-1/, ...
        final/
          full-page.html      ← final state after all applied patches
```

Each audit is fully self-contained — you can diff two timestamps for the same post to see what changed between runs.

Published audits also mirror to `~/Desktop/audits/reports/` by default. Pass `--no-publish-repo` to skip the git copy.
